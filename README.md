# taobao-category-selection

**一句话分析淘宝任意品类——在 Claude Code 中说出品类名，自动完成数据采集、五维评分、生成报告、撰写深度分析。全程无需手动操作。**

## 能力概览

| 指标 | 数值 |
|------|:---:|
| 单次分析产品数 | 89-120 个（2页串行抓取） |
| 评分维度 | 5 维 100 分制 |
| 输出格式 | Markdown + Excel 12 Sheets + HTML 仪表板 |
| Claude 深度分析 | 7 节完整分析（执行摘要→卖家画像） |
| 平均分析时间 | 约 60 秒 |
| 数据来源 | 淘宝/天猫实时搜索 |
| 反爬风险 | 零（复用真实 Chrome 登录态） |

## 安装

```bash
git clone https://github.com/didiwaiwai/taobao-category-selection.git
cd taobao-category-selection
bash setup.sh
```

前置条件：Node.js 18+、Python 3.9+、Chrome 浏览器、Claude Code。

首次使用需扫码登录淘宝：
```bash
bb-browser daemon start
bb-browser open https://www.taobao.com
```

## 使用

在 Claude Code 中（cd 到项目目录），说出品类名即可：

```
分析淘宝男士素颜霜
```

Claude Code 自动执行五步：采集数据 → 五维评分 → 生成报告 → 撰写深度分析 → 注入完整报告。

**输出目录：** `taobao-reports/{品类}_{日期}/`

```
taobao-reports/男士素颜霜_20260701/
├── report.md          # Markdown 完整报告（含 Claude 深度分析）
├── report.xlsx        # Excel 12 Sheets（含图表）
├── dashboard.html     # Chart.js 可视化仪表板
├── data.json          # 原始评分数据
└── data_brief.md      # LLM 上下文简报
```

也可手动使用：
```bash
bb-browser site taobao/category-score 蓝牙耳机 --json > data.json
python scripts/generate_taobao_report.py data.json
```

## 五维评分模型

每个品类基于五个维度量化评估，满分 100 分。

| 维度 | 分值 | 计算指标 | 说明 |
|------|:---:|------|------|
| 市场规模 | 20 | Top产品预估月销量 | 五档：50万+/20万+/10万+/5万+/其他 |
| 增长潜力 | 25 | 自适应三层新品占比 + 页间动态性 | 上市时间 > 首单价 > 标题新品 > C店兜底；Dynamism Index 页间交叉验证 |
| 竞争烈度 | 20 | 店铺CR3 + HHI + 同类商品数 | 双HHI兜底（品牌数据充足时用品牌HHI，否则自动降级为店铺HHI） |
| 进入壁垒 | 20 | 天猫占比 × 新品活跃度 4×3矩阵 + 补贴折价 | 12种壁垒类型（开放繁荣/品牌换皮/品牌锁定/封闭市场等） |
| 利润空间 | 15 | 平均售价 + 1688成本对标 | 四档：¥200+/¥100+/¥40+/其他 |

**评级：** 80+ 优秀 · 70+ 良好 · 50+ 一般 · <50 较差

## 核心优势

### 自适应三层数据源

不同品类有不同数据可用，模型自动选择最优指标：

- **Tier 1** — 上市时间（3C数码品类可提取精确上架日期，直接衡量新品活跃度）
- **Tier 2** — 首单价 + 标题新品词（行为信号比声明属性更可靠，含品牌换皮折扣机制）
- **Tier 3** — C店占比 + 热销爆款（所有品类兜底，上限压制 30% 防止过度乐观）

### 4×3 交叉壁垒矩阵

不是简单的"天猫占比高=壁垒高"。天猫占比和新品活跃度是两个独立维度，交叉后得到 12 种市场画像：

| 天猫占比 \ 新品活跃度 | 高(>20%) | 中(10-20%) | 低(<10%) |
|---------------------|:---:|:---:|:---:|
| 低(<30%) | 开放繁荣 20分 | 门槛低 18分 | 静水市场 14分 |
| 中(30-70%) | 品牌换皮 14分 | 较高壁垒 12分 | 品牌锁定 8分 |
| 高(>70%) | 品牌迭代 12分 | 高壁垒 8分 | 封闭市场 6分 |

例如：天猫 80% 但新品 81% = "品牌迭代市场"（12分），天猫 80% 但新品 4% = "封闭市场"（6分）——同样天猫占比，壁垒评分差一倍。

### Dynamism Index 页间动态性

通过对比搜索结果第1页和第2页的产品重叠度（Jaccard 相似度），量化市场的活跃程度：

- **>0.5 高动态**：页面间产品差异大，市场在快速轮换，新品有被平台推荐的机会
- **0.3-0.5 中等动态**：市场有一定周转但相对稳定
- **<0.3 低动态**：页面间产品高度重合，市场固化

DI 作为增长潜力的修正项（±2分），帮助区分"低新品率但市场活跃"和"低新品率且市场僵化"两种完全不同的情况。

### Claude Code 端到端自动化

项目内嵌完整的 Claude Code Skill（`.claude/skills/taobao-category/SKILL.md`），进入项目目录后自动激活。一次对话完成：

1. 自动检测并启动 daemon
2. 采集数据 + 五维评分（适配器在浏览器端实时计算）
3. 生成 Markdown + Excel + HTML 报告
4. 读取数据简报
5. 以选品运营专家身份撰写 7 节深度分析
6. 注入完整报告

**分析不会混入其他品类信息**——Skill 机制每次独立加载，上下文干净。

## 技术原理

### 数据采集

复用淘宝页面内置的 `window.lib.mtop` 库。请求从真实 Chrome 发出，IP/Cookie/浏览器指纹完全正常，HMAC 签名由淘宝 JS 自动生成。淘宝感知到的是正常用户浏览行为，不是爬虫。

### 串行抓取 + 限流保护

页间延迟 3 秒串行请求，超时 45 秒，避免触发淘宝反爬。

### 浏览器端评分计算

五维评分在 adapter（浏览器端 JS）中实时计算，Python 端只做报告格式化。评分逻辑与数据采集在同一环境中，无数据传输损耗。
