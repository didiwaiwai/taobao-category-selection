# 淘宝/天猫品类选品分析工具

> **一句话分析淘宝任意品类——自动采集数据、五维评分、1688供应链对标、AI深度分析、生成三格式完整报告。全程无需手动操作浏览器。**

## 为什么值得用

| 痛点 | 传统方式 | 本工具 |
|------|------|------|
| 搜数据 | 运营手动翻几十页淘宝 + 1688 比价 | 2分钟自动采集 180条1688 + 96件淘宝 |
| 做判断 | 凭经验拍脑袋，缺乏量化依据 | **五维 100 分制评分** + 品牌 HHI + DI 动态指数 |
| 看竞品 | 人工点开 Top 产品看标题和评价 | **AI 逐个拆解 Top 5** 的标题策略、定价逻辑、成功因素 |
| 算利润 | Excel 手动填，忘记算退货/手续费 | **利润计算器**自动填默认值 + 可调参数 + 实时重算 |
| 写标题 | 凭感觉堆关键词 | **AI 生成 3 个 SEO 优化标题**（品牌型/种草型/搜索型） |
| 找供应链 | 1688 一页页翻 | mtop API **180 条批发数据**，价格分布一目了然 |

## 安装

```bash
git clone https://github.com/didiwaiwai/taobao-category-selection.git
cd taobao-category-selection
bash setup.sh
```

前置条件：Node.js 18+、Python 3.9+、Chrome 浏览器、Claude Code。

首次使用需载入淘宝页面（加载 mtop 库）：
```bash
bb-browser daemon start
bb-browser open https://www.taobao.com
```

## 使用

在 Claude Code 中（cd 到项目目录）说出品类名：

```
分析钛金属炊具
```

自动执行：淘宝采集 → 1688采集 → 五维评分 → 生成报告 → AI深分析 → 注入完整报告。

**输出目录：** `taobao-reports/{品类}_{日期}/`

```
taobao-reports/钛金属炊具_20260704/
├── report.md          # Markdown 完整报告（含 9 节 AI 深度分析）
├── report.xlsx        # Excel 14 Sheets（含利润计算器+竞品分析）
├── dashboard.html     # 13 标签页仪表板（图表+数据+利润计算器+AI分析）
├── data.json          # 原始淘宝评分数据
├── data_1688.json     # 1688 批发数据（180条）
├── data_brief.md      # LLM 上下文简报（含卖点+1688产品列表）
└── llm_analysis.md    # AI 深度分析
```

也可手动使用：
```bash
bb-browser site taobao/category-score 蓝牙耳机 --json > data.json
python scripts/collect_1688.py 蓝牙耳机
python scripts/generate_taobao_report.py data.json
```

## 核心亮点

### 1. 1688 与淘宝同架构 mtop API 采集

不是爬 DOM，而是调用阿里内部 `mtop.relationrecommend.WirelessRecommend.recommend` API（1688 用 appId 32517，淘宝用 34385）。单次 180 条产品、100% 关键词匹配、零验证码。

### 2. 品牌 HHI + DI 动态指数

- **品牌 HHI**：从 `item.brandName` 多源提取，覆盖率 34%+，真实品牌集中度（非店铺兜底）
- **DI 动态指数**：页间 Jaccard 相似度量化市场活跃度（DI=1.00 为最高，如焗油梳子/男士叶酸精华）

### 3. AI 竞品深度分析

不是泛泛的 SWOT，而是 **逐个拆解 Top 5 产品的标题策略、定价逻辑、成功因素、可复制差异化**。末尾输出竞争格局总结表。

### 4. 可调参数利润计算器

自动填入 1688 中位批发价和淘宝均价，支持手动调整：快递、包装、推广费、退货率、手续费、运险费。JavaScript 实时重算毛利率、净利率。

### 5. AI 优化标题模板

基于 Top 产品标题关键词分析，生成 3 个 SEO 优化标题：
- **标题 1**：品牌旗舰型（适合旗舰店）
- **标题 2**：场景种草型（适合小红书/抖音）
- **标题 3**：长尾搜索型（最大化搜索曝光）

每个标题标注核心关键词、策略说明和定价建议。

### 6. 4×3 交叉壁垒矩阵

天猫占比 × 新品活跃度 → 12 种市场画像（开放繁荣/品牌换皮/品牌锁定/封闭市场等）。相同天猫占比，新品活跃度不同，壁垒评分差一倍。

## 五维评分模型

| 维度 | 分值 | 计算指标 |
|------|:---:|------|
| 市场规模 | 20 | 预估月销量（五档 50万+/20万+/10万+/5万+/其他） |
| 增长潜力 | 25 | 新品占比 + DI 动态指数 + 品牌换皮修正 |
| 竞争烈度 | 20 | 店铺 CR3 + 品牌 HHI + 同类商品数 |
| 进入壁垒 | 20 | 天猫占比 × 新品活跃度 4×3 矩阵 + 补贴折价 |
| 利润空间 | 15 | 均价 + 1688 成本对标 |

**评级：** 80+ 优秀 · 70+ 良好 · 50+ 一般 · <50 较差

## 项目结构

```
taobao-category-selection/
├── bb-sites/                          # bb-browser 适配器（项目副本）
│   ├── 1688/offer-score.js            # 1688 mtop API 采集
│   └── taobao/                        # 淘宝适配器
│       ├── keyword-suggest.js         # 关键词建议
│       └── product-deepdive.js        # 竞品深挖
├── scripts/                           # Python 报告生成
│   ├── generate_taobao_report.py      # 核心：三格式报告 + 利润计算器
│   ├── collect_1688.py                # 1688 数据采集
│   ├── collect_keywords.py            # 关键词采集
│   └── taobao_score.py               # 控制台评分格式化
├── .claude/skills/taobao-category/    # Claude Code Skill
└── CLAUDE.md                          # 项目指令
```

> **注意**：淘宝和 1688 的适配器（`category-score.js`、`offer-score.js`）需要安装到 `~/.bb-browser/sites/` 才能工作。运行 `setup.sh` 或手动复制。
